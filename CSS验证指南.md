## 🔍 如何验证CSS是否生效

### 方法1：查看首页测试区域（最简单）

1. 访问：https://warmearthhome.com
2. 滚动到页面底部
3. 查看"CSS样式验证测试区域"

**检查项目：**

#### ✅ 1. 主按钮测试
- **应该看到**：暖棕色按钮（#D4A574）
- **悬停效果**：按钮颜色变深，轻微上浮
- **验证**：按钮背景色应该是 `rgb(212, 165, 116)` 或 `#D4A574`

#### ✅ 2. 次按钮测试
- **应该看到**：透明背景，暖棕色边框
- **悬停效果**：填充暖棕色背景

#### ✅ 3. 信任提示条
- **应该看到**：灰色背景框，2列布局
- **包含**：图标 + 文字（澳规插头/RCM认证、30天退换/220-240V）

#### ✅ 4. 参数表格
- **应该看到**：清晰的表格，表头灰色背景
- **结构**：完整的thead和tbody

#### ✅ 5. 分类卡片
- **应该看到**：3个白色卡片，每个包含图片、标题、描述
- **悬停效果**：鼠标悬停时卡片上浮，阴影加深

#### ✅ 6. 服务承诺条
- **应该看到**：浅灰色横条，居中文字
- **内容**：Free Shipping / 30-Day Returns / RCM Certified / AU Plug Ready

### 方法2：使用浏览器开发者工具

1. 按 `F12` 打开开发者工具
2. 切换到"Elements"标签
3. 选择一个测试按钮（如"Add to Cart"）
4. 在右侧"Styles"面板查看计算后的样式
5. 检查 `background-color` 是否为 `rgb(212, 165, 116)`

### 方法3：检查CSS文件

1. 在开发者工具中切换到"Sources"或"Network"标签
2. 刷新页面
3. 查找 `custom-css` 或 `additional-css` 相关的CSS文件
4. 确认CSS代码已加载

## 🐛 如果CSS未生效

### 可能的原因：

1. **缓存问题**
   - 清除浏览器缓存（Ctrl+Shift+Delete）
   - 清除WordPress缓存（LiteSpeed Cache）

2. **CSS未正确保存**
   - 检查WordPress后台 > 外观 > 自定义 > 额外CSS
   - 确认CSS代码存在

3. **CSS优先级问题**
   - 检查是否有其他CSS覆盖
   - 在开发者工具中查看样式冲突

### 解决方法：

1. **清除缓存**
   ```
   WordPress后台 > LiteSpeed Cache > 清除全部缓存
   ```

2. **强制刷新**
   - 按 `Ctrl+F5` 强制刷新页面

3. **检查CSS代码**
   - 确认CSS代码已正确添加到"额外CSS"
